How to Use

1. Connect to your TA612C

USB (Chrome/Edge only)
Click USB and select your serial port. Uses the Web Serial API for direct hardware access — no extra software needed. Serial config: 9600 baud, 8N1.
Bridge (any browser)
Start the bridge: uv run bridge.py, then click Bridge. This relays binary data over WebSocket so Firefox, Safari, and remote browsers can talk to the device.
Linux: device not showing up?
The TA612C uses a CH340 USB-to-serial chip. On Ubuntu/Debian, the brltty package claims this chip and blocks the driver from creating /dev/ttyUSB0. Fix: sudo apt remove brltty, then unplug and replug. You also need to be in the dialout group: sudo usermod -aG dialout $USER (log out and back in after).

2. Toolbar buttons

Play / Stop
Start or stop automatic polling. Sends the real-time data command at the configured interval (default 1000 ms). The device responds with all 4 channel temperatures each time.
Record / Save
Record captures every reading with a timestamp. Save stops and downloads a CSV file with columns: Timestamp, T1, T2, T3, T4.
Sync Clock
Synchronize the device clock to your PC time. Sends the current date/time in BCD format.

3. Chart

4 colored lines show all thermocouple channels simultaneously. Click legend entries to show/hide individual channels. Window sets how much history to show. Y Min/Max lock the vertical axis. Clear erases all data and resets statistics.

4. Demo mode

Click Demo to generate fake 4-channel thermocouple data and try out the interface without hardware.

5. Typical workflow

Connect → Record → Play → watch live temperatures and chart → Stop → Save to download CSV.

Running locally

Don't open index.html directly — ES modules need a web server. Run uv run serve.py and open http://localhost:8000.

      TA612C Protocol Reference
      

        
Commands (PC → Device)

        

          
            
CommandHexDescription

          
          
            
Start Real-timeAA 55 01 03 03Request one real-time reading (4 channels)

            
StopAA 55 00 03 02Stop transmitting, returns model + firmware version

            
Start Logged DataAA 55 02 03 04Download recorded data from device memory

            
Time SyncAA 55 03 07 <BCD> sumSynchronize device clock (7 BCD-encoded bytes)

          
        


        
Response Frames (Device → PC)

        

          
            
TypeCmdPayload

          
          
            
Real-time Data014 × 16-bit LE temperatures (value ÷ 10 = °C)

            
Model/Version00Model ID (16-bit LE) + Version (16-bit LE, ÷100)

            
Record Data02Groups of 4 × 16-bit LE channel values

          
        


        
Frame Structure

        
All frames: Header (2 bytes) + Command (1) + Length (1) + Payload + Checksum (1). PC→Device header: AA 55. Device→PC header: 55 AA. Checksum = low byte of sum of all preceding bytes.


        
Serial Config

        
9600 baud, 8 data bits, 1 stop bit, no parity, no handshake. RS232C interface.
